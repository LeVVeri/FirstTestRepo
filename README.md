# Краткий конспект теории

Краткий конспект теории
Диаграмма деятельности (Activity Diagram)
Предназначена для моделирования процесса выполнения операций и представляет алгоритм варианта использования.
Основные элементы:
Состояние действия — прямоугольник со скругленными углами, содержит выражение действия (глагол с пояснениями)
Начальное состояние — закрашенный круг (в верхней части диаграммы)
Конечное состояние — круг с ободком (в нижней части)
Переходы — сплошные линии со стрелками (нетриггерные, срабатывают сразу после завершения действия)
Ветвление — ромб без текста, условия перехода указываются в квадратных скобках [условие]
Разделение/слияние — горизонтальная черта для параллельных потоков
Диаграмма состояний (Statechart Diagram)
Описывает последовательность состояний и переходов объекта в течение его жизненного цикла. Используется для моделирования реактивных объектов.
Основные элементы:


-H:"avito.ru dns-shop.ru" -An -d1 -d3+s -s6+s -d9+s -s12+s -d15+s -s20+s -d25+s -s30+s -d35+s -r1+s -S -a1 -As -d1 -d3+s -s6+s -d9+s -s12+s -d15+s -s20+s -d25+s -s30+s -d35+s -S -a1
Состояние — прямоугольник со скругленными углами, разделен на две секции: имя состояния (глагол/причастие: звенит, занят) и список внутренних действий
Переход — сплошная линия со стрелкой между состояниями
Событие — стимул, вызывающий срабатывание перехода (указывается у перехода)
Сторожевое условие — булевское выражение в квадратных скобках [условие]
Диаграмма классов
Показывает статическую структуру системы: классы, их атрибуты, операции и связи между классами (ассоциация, агрегация, композиция, наследование).
